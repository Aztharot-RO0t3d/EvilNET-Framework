# EvilNet
**by Aztharot**

## Origin

This project didn’t start in a lab.
It started in a pizza place.

A simple question came up:
**how easy is it to impersonate a Wi-Fi network?**

From there, the idea escalated: deploy an Evil Twin, replicate its identity, build parallel infrastructure, and observe how far trust actually goes.

Not as an attack, but as a proof.

---

## What is EvilNet?

EvilNet is a framework focused on simulating Evil Twin scenarios in controlled environments.

It is built for practical use:
* experimenting with real setups
* validating assumptions
* understanding network and user behavior
* improving defensive thinking through offensive practice

---

## How it works

EvilNet orchestrates:
* A rogue access point with replicated identity
* A secondary interface providing real connectivity
* Routing and NAT rules
* Traffic manipulation at network level
* DNS redirection to internal services
* Controlled portal exposure

The result is a functional environment that mimics real-world conditions while allowing observation and control.

---

## Purpose

* Practice Wi-Fi offensive techniques
* Understand real attack vectors without abstraction
* Evaluate trust and behavior in networks
* Build practical cybersecurity skills
